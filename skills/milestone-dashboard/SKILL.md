# Milestone and open-loop dashboard

Represent commitments with stable identifiers, owner, status, timing,
dependency, evidence, and a clear next action. Keep assumptions visibly
separate from verified facts. Open questions should use a structured schema,
not freeform strings, so a deterministic scanner can prioritize them.

Use a targeted question to close a high-impact unknown, then update the
canonical milestone through the approved state adapter. Do not manufacture
progress from a dashboard click, duplicate a cancelled event, or turn a
generated projection into canonical truth.
