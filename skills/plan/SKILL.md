# Minimal planning

Use planning when the task has multiple dependent decisions or a meaningful
implementation risk. Start from ground truth, identify the smallest viable
change, and make success criteria observable. Prefer a reversible spike when
the unknown is technical and a short plan when the work is already understood.

Do not use a plan to create ceremony, speculative abstractions, or a long list
of unverified future work. Revisit the plan after a failed check or material
new evidence, then keep the final plan and implementation aligned.
